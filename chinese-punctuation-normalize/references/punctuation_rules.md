## 基本转换规则

### 标点符号映射表

| 英文标点 | 中文标点 | Unicode | 说明 |
|---------|---------|---------|------|
| `,` | `，` | U+FF0C | 逗号 |
| `.` | `。` | U+3002 | 句号 |
| `:` | `：` | U+FF1A | 冒号 |
| `?` | `？` | U+FF1F | 问号 |
| `!` | `！` | U+FF01 | 感叹号 |
| `;` | `；` | U+FF1B | 分号 |
| `(` | `（` | U+FF08 | 左括号 |
| `)` | `）` | U+FF09 | 右括号 |
| `"` | `“”` | 左引号 U+201C, 右引号 U+201D | 双引号 |
| `'` | `‘’` | 左引号 U+2018, 右引号 U+2019 | 单引号 |

### 引号转换规则

英文双引号 `"` 需要成对转换为中文引号:
- 左引号: `“` (U+201C)
- 右引号: `”` (U+201D)

英文单引号 `'` 需要转换为中文单引号:
- 左引号: `‘` (U+2018)
- 右引号: `’` (U+2019)

**规则**: 按照出现顺序,奇数次为左引号,偶数次为右引号。

## 特殊场景处理

### 1. 不转换的场景

#### 1.1 代码相关

**多行代码块**
````markdown
```python
def hello():
    print("Hello, World!")  # 这里的标点保持不变
```
````

**行内代码**
```markdown
使用 `function(a, b)` 调用函数
```

**数学公式和函数语法**
```markdown
计算 f(x) = sum(a,b) 的值
```

#### 1.2 URL 和链接

**纯 URL**
```markdown
访问 https://example.com/path?id=1&name=test
```

**Markdown 链接**
```markdown
参见 [文档](https://docs.example.com/api?v=2)
```
注意: Markdown 链接中的圆括号保持英文,因为它们是语法的一部分。

#### 1.3 英文内容

**完整英文句子**
```markdown
This is a complete English sentence. It should keep English punctuation.
```

**专有名词和品牌名**
```markdown
使用 iPhone, macOS, Windows 10 等产品
```

#### 1.4 数字格式

**小数点**
```markdown
圆周率约等于 3.14
```

**千分位分隔符**
```markdown
世界人口约 7,800,000,000
```

### 2. 需要转换的场景

#### 2.1 中文文本中的标点

```markdown
这是一段中文文本,需要转换为中文标点。
函数说明:该方法接收两个参数,返回布尔值。
```

转换后:
```markdown
这是一段中文文本，需要转换为中文标点。
函数说明：该方法接收两个参数，返回布尔值。
```

#### 2.2 标题和说明文字

```markdown
## 参数说明:用户配置
配置文件路径: /etc/app/config.yml
```

转换后:
```markdown
## 参数说明:用户配置
配置文件路径：/etc/app/config.yml
```

注意: Markdown 标题标记 `##` 本身不变,但后面的冒号要转换。

#### 2.3 并列词语

```markdown
支持的语言:Python,Java,Go,JavaScript
```

转换后:
```markdown
支持的语言：Python、Java、Go、JavaScript
```

**规则**: 并列的词语(特别是中文词语)之间使用顿号 `、` 而非逗号 `，`。

## 常见错误示例

### 错误 1: 英文引号混用

**错误**
```markdown
他说"这是一个测试"。
```

**正确**
```markdown
他说"这是一个测试"。
```

### 错误 2: URL 中使用了中文标点

**错误**
```markdown
访问 https://example.com/api？id=1&name=test
```

**正确**
```markdown
访问 https://example.com/api?id=1&name=test
```

### 错误 3: 代码块中转换了标点

**错误**
````markdown
```python
def test（）：
    print（"hello"）
```
````

**正确**
````markdown
```python
def test():
    print("hello")
```
````

### 错误 4: 数字小数点被转换

**错误**
```markdown
版本号 2。0 发布
价格 99。99 元
```

**正确**
```markdown
版本号 2.0 发布
价格 99.99 元
```

### 错误 5: Markdown 链接括号被转换

**错误**
```markdown
参见[文档]（https://example.com）
```

**正确**
```markdown
参见[文档](https://example.com)
```

## 边界情况

### 1. 混合中英文

**情况**
```markdown
Python 是一门流行的语言,广泛应用于 AI 领域。
```

**处理**: 中文部分的标点转换为中文标点。

**结果**
```markdown
Python 是一门流行的语言，广泛应用于 AI 领域。
```

**中英数字间距**
```markdown
我是abcd小王子,你是123,他是kkk.
```

**处理**: 中文与英文/数字直接相邻时补充半角空格，但不猜测原文缺失的语义停顿。

**结果**
```markdown
我是 abcd 小王子，你是 123，他是 kkk。
```

### 2. 列表编号

**数字列表**
```markdown
1. 第一项
2. 第二项
3. 第三项
```

**处理**: 列表编号后的点 `.` 保持英文,这是 Markdown 语法的一部分。

**中文序号列表**
```markdown
第一,准备工作
第二,执行操作
```

**处理**: 中文序号后的逗号应该转换,但作为列表标识时,可以使用顿号或冒号。

**结果**
```markdown
第一、准备工作
第二、执行操作
```

或:

```markdown
第一：准备工作
第二：执行操作
```

### 3. 括号内的标点

**情况**
```markdown
详见用户手册(第三章,第二节)
```

**处理**: 括号和内部标点都转换为中文。

**结果**
```markdown
详见用户手册（第三章，第二节）
```

### 4. 引号嵌套

**情况**
```markdown
他说:"老师告诉我们'要认真复习'。"
```

**处理**: 外层用双引号,内层用单引号。

**结果**
```markdown
他说：“老师告诉我们‘要认真复习’。”
```

**注意**: 中文单引号为 `''` (U+2018/U+2019)。

### 5. 特殊符号

**省略号**
- 英文: `...`
- 中文: `……` (两个中文省略号 U+2026)

**破折号**
- 英文: `--` 或 `-`
- 中文: `——` (两个中文破折号 U+2014)

**连接号**
- 英文: `-`
- 中文: `-` (短横线) 或 `—` (一字线) 或 `~` (波浪线),根据具体语境

### 6. 表格中的标点

**情况**
```markdown
| 项目 | 说明 |
|------|------|
| 版本 | 1.0 |
| 价格 | 99.99元 |
```

**处理**: 
- 表格分隔符 `|` 和 `---` 不变
- 单元格内的标点按照中文规则转换
- 数字中的小数点保持不变

**结果**
```markdown
| 项目 | 说明 |
|------|------|
| 版本 | 1.0 |
| 价格 | 99.99元 |
```

### 7. 代码注释中的中文

**情况**
````markdown
```python
# 这是一个注释,说明代码功能
x = 1  # 设置变量值
```
````

**处理**: 代码块内的所有内容(包括注释)保持原样,不做转换。

**原因**: 代码块的边界非常明确,注释也是代码的一部分。

### 8. HTML 标签属性

**情况**
```markdown
<div style="color:red,font-size:14px">文本</div>
```

**处理**: HTML 标签内的属性值保持英文标点。

**结果**
```markdown
<div style="color:red,font-size:14px">文本</div>
```

**原因**: HTML 是结构化语言,标点符号是语法的一部分。

## 检查清单

在转换完成后,检查以下项目:

- [ ] 代码块(包括行内代码)未被修改
- [ ] URL 链接未被修改
- [ ] Markdown 链接语法正确
- [ ] 数字小数点和千分位保持不变
- [ ] 英文句子保持英文标点
- [ ] 并列词语使用顿号
- [ ] 引号成对出现,方向正确
- [ ] 中文与英文/数字之间的空格已补齐
- [ ] 未擅自补充原文不存在的语义标点
- [ ] 列表编号格式正确
- [ ] 表格结构未被破坏

### 并列词语与括号

**修改前**：苹果,香蕉,橙子（三种水果）
**修改后**：苹果、香蕉、橙子（三种水果）

**不修改**：`` `const x = { a: 1, b: 2 };` `` 和代码块内的所有内容
